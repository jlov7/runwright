# Onboarding Funnels And First Success Moments (M4)

## Purpose
Define persona-based onboarding funnels and explicit first success outcomes for the web shell.

## Funnel A: New Player (Fast Start)
1. Create profile.
2. Run tutorial hint.
3. Save progress.
4. Publish first level.
First success moment:
- Banner: "Celebration: First success achieved."
- Next recommendation: start campaign chapter.

## Funnel B: Returning Player (Resume)
1. Restore persisted onboarding snapshot.
2. Review checklist + diagnostics.
3. Resume from next incomplete step.
First success moment:
- Checklist reaches all required milestones.
- Dashboard status changes to `Ready`.

## Funnel C: Unsure Player (Skip/Resume)
1. Skip onboarding temporarily.
2. Continue exploring dashboard/help.
3. Resume checklist with one action.
First success moment:
- Resume action returns to onboarding center with next step preselected.

## Funnel D: Guided Demo (Bootstrap)
1. Load sample progress.
2. Validate campaign unlock.
3. Start campaign chapter.
First success moment:
- Campaign surface moves from blocked to chapter started.

## Failure Recovery Funnels
- Tutorial blocked: diagnostics prompt retry tutorial.
- Save conflict: diagnostics prompt save retry.
- Publish blocked: diagnostics prompt valid level publish.
- Campaign blocked: diagnostics prompt publish prerequisite.
- Ranked rejection: diagnostics keep onboarding/campaign progress unchanged.

# Release Signing Runbook

## Scope
- Applies to signed release bundles produced by `skillbase export --sign-private-key`.
- Covers key generation, CI secret setup, verification evidence, and rotation.

## Key generation
```bash
openssl genpkey -algorithm ed25519 -out bundle-private.pem
openssl pkey -in bundle-private.pem -pubout -out bundle-public.pem
```

## CI secret provisioning
- Configure GitHub Actions repository secrets:
- `SKILLBASE_RELEASE_PRIVATE_KEY`: full PEM content of `bundle-private.pem`
- `SKILLBASE_RELEASE_PUBLIC_KEY`: full PEM content of `bundle-public.pem`

## Verification evidence
- Workflow: `.github/workflows/release-verify.yml`
- Deterministic export mode is required (`--deterministic` with fixed `SOURCE_DATE_EPOCH`).
- Tagged releases must use signed annotated tags (`release:tag:verify` must report `ok: true`).
- Required artifacts per release verification run:
- `skillbase-release.zip`
- `SHA256SUMS`
- `skillbase-release.verify.json`
- `tag-signature.verify.json`
- GitHub attestation generated by `actions/attest-build-provenance` for `skillbase-release.zip`
- Downloaded attestation bundle file: `skillbase-release.attestation.jsonl`
- Release verification scorecard artifacts: `release-scorecard.json`, `release-scorecard.md`
- Release evidence verdict artifact: `release-scorecard.verify.json`
- Immutable artifact manifest artifacts: `release-artifact-manifest.json`, `release-artifact-manifest.verify.json`
- Release note artifact: `release-notes.md`
- `skillbase-release.verify.json` must report:
- `integrityOk: true`
- `signatureVerified: true`

## Consumer verification command
```bash
bash scripts/verify_release_consumer_artifact.sh \
  --artifact .release/project/skillbase-release.zip \
  --checksums .release/project/SHA256SUMS \
  --verify-json .release/project/skillbase-release.verify.json \
  --public-key .release/keys/public.pem \
  --repo owner/repo \
  --bundle .release/project/skillbase-release.attestation.jsonl
```

## Key identity binding
- `_bundle/manifest.json` includes `signature.keyId`.
- `keyId` is `sha256` over public key SPKI DER bytes.
- Verification fails if `signature.keyId` does not match the supplied public key.

## Rotation procedure
1. Generate a fresh ed25519 key pair.
2. Update `SKILLBASE_RELEASE_PRIVATE_KEY` and `SKILLBASE_RELEASE_PUBLIC_KEY` secrets atomically.
3. Trigger `release-verify` workflow and confirm:
- bundle verification success
- attestation verification success
- tag signature verification success (for tagged releases)
- immutable artifact manifest verification success
- release scorecard reports all checks as `success`
- release evidence verification reports `ok: true`
4. Archive the previous public key in release notes/audit storage for historical validation.

## Incident response
1. If key compromise is suspected, rotate keys immediately.
2. Re-run release verification and regenerate bundle artifacts.
3. Treat previous artifacts as untrusted until re-verified with approved public keys.
